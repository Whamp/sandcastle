## What is Sandcastle?

Sandcastle is a TypeScript toolkit for host-first task coordination. The default happy path selects a ready GitHub Issue-backed Task, executes an agent directly on the host with worktree safety, lands the change, and closes the Issue-backed task after land succeeds.

Sandboxed execution remains available as a secondary execution mode when you want Docker, Podman, Vercel, or a custom provider.

## Key concepts

- **Task Coordination** — selects ready tasks, claims them, executes them, lands changes, and closes their backlog artifacts
- **Task vs Issue** — **Task** is the canonical backlog term; a GitHub **Issue** is the first backlog-specific representation
- **Execution mode** — host execution is the default happy path; sandboxed execution remains optional
- **Backlog adapter** — the integration that exposes tasks from one backlog system such as GitHub Issues
- **Agent provider** — the coding-agent integration Sandcastle invokes for a run

## Quick start

### 1. Install

```bash
npm install @ai-hero/sandcastle
```

### 2. Initialize

Run `sandcastle init` in your repository. The default happy path scaffolds a Pi-first, host-first GitHub Issue Task Coordination worker.

```bash
npx sandcastle init
```

### 3. Configure env vars

```bash
cp .sandcastle/.env.example .sandcastle/.env
```

Fill in the required values for your chosen agent and backlog adapter.

### 4. Run the scaffolded worker

```bash
npx tsx .sandcastle/main.mts
```

If your project's `package.json` uses `"type": "module"`, the scaffolded file will be `main.ts` instead.
