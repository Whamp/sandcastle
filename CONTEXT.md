# Sandcastle Context Entry Point

This repo now uses a context map because its language spans two distinct contexts: upstream **Backlog Curation** and downstream **Task Coordination**.

Start with [CONTEXT-MAP.md](./CONTEXT-MAP.md).

- **Backlog Curation**: discovery conversation → shared understanding → PRD → task breakdown → promotion to ready
- **Task Coordination**: ready task selection → claim lease → execution → land → close or record outcome

Avoid treating these as one flat model.

The typical backlog curation flow behind this repo is discovery interview, PRD synthesis, and task or issue breakdown before automated task coordination begins.
